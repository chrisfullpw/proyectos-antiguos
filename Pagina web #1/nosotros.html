FIX.NOTE NOSOTROS

INTRODUCCION FIX.NOTE

El presente proyecto se lleva a cabo por medio de un caso de estudio el cual proporciona la información necesaria para su debido desarrollo. Actualmente se ve el uso de papel dentro de las instituciones educativas (listados con notas estudiantiles y/o horarios) para llevar a cabo la información, la manera en la que lo manejan es desorganizada y poco eficaz, por esta razón se decide diseñar un sistema de información que facilite flujo de datos y la información usada por coordinadores, docentes y estudiantes brindando un servicio eficaz para poder lograr una disminución en el uso de papel dentro de las instituciones educativas. Tenemos como propósito el implementar este sistema de información para las diferentes instituciones educativas, este sistema de información se procederá a desarrollar en cuatro etapas o fases que involucran los diferentes procesos de creación de software, permitiendo, lograr alcanzar el objetivo propuesto; La primera etapa consiste en el análisis de investigación y la necesidad propuesta por la institución para la creación del aplicativo, en esta fase se va a involucrar el levantamiento de información, análisis de requerimientos funcionales y no funcionales, delimitación y alcance del aplicativo.

QUIENES SOMOS

FIX.NOTE, es un sistema de información para la gestión de datos de entidades educativas, con recursos útiles teniendo en cuenta los módulos con diferentes características como lo son:

  • La entrada de información
  • El procesamiento de datos
  • El almacenamiento de la información
  • La salida de información
  • El análisis de datos
  • La visualización de los datos.

QUE ES

Es la aplicación web más importante que hemos hecho como equipo desarrollador para el consumo y mejoramiento de las entidades educativas, por lo cual las instituciones educativas, se les permitirá manejar con más facilidad todas las bases de datos que tengan en el colegio y para los docentes un archivamiento de toso los procesos que lleva en la institución.

PORQUE SE CREO

En base a las problemáticas que se plantean por parte de las instituciones educativas, el proyecto quiere facilitarle y hacer más eficaz la actualización de datos y el uso de estos al docente, se debe tener en cuenta que el docente en sus horas laborales o usa un computador para llamar a lista Todo esto por medio de una aplicación de escritorio y/o aplicación móvil la cual permita la gestión y administración de datos que sean ingresados a esta.














MISIONES

MISION: Es generar un orden a la hora de buscar información. El S.I le brinda al usuario la eficacia a la hora de buscar información en la base de datos permitiéndole al usuario un mejor desempeño en su trabajo, gestionra las bases de datos del personal de las entidades educativas y visualización de información básica para los estudiantes.

VISION: El diseñar los diagramas del modelo entidad relación, el análisis de arquitectura, definición del lenguaje de programación, el motor de base de datos a utilizar, la creación de la base de datos, la respectiva parametrización de los datos básicos y la configuración inicial, cumpliendo los requisitos de hardware y software requeridos por la entidad educativa, y que estanunca valla a generar problems y sea muy efectiva en la hora de ejecitar los programas y el motor,se tine pensado que este proyecto estara para el 2023.
